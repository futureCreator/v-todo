# AI 주간 리뷰 (Weekly Review) 설계

## 개요

매주 일요일 밤, 해당 주(월~일)의 데일리 노트를 Gemini에 보내 **인사이트를 불릿 포인트로 추출**하고, `data/notes/weekly-review/YYYY-WNN.md` 파일로 저장한다. 사용자는 기존 범용 노트 UI의 `weekly-review` 폴더에서 바로 열람할 수 있다.

## 데이터 흐름

1. **트리거**: 매주 일요일 밤 cron 자동 실행 (수동 실행 없음)
2. **입력 수집**:
   - `data/daily-notes/`에서 해당 주 월~일 7개 파일 읽기 (없는 날은 건너뜀)
   - `data/notes/weekly-review/`에서 직전 주 리뷰 파일 읽기 (없으면 생략)
3. **Gemini 호출**: 데일리 노트 + 이전 주 리뷰 + 프롬프트 → 인사이트 불릿 생성
4. **파일 저장**: `data/notes/weekly-review/YYYY-WNN.md`
5. **열람**: 범용 노트 UI의 `weekly-review` 폴더에서 확인

## API

### `POST /api/ai/weekly-review`

- 요청 시 해당 주(월~일) 데일리 노트를 자동 수집
- 직전 주 리뷰 파일도 함께 읽어서 프롬프트에 포함
- 노트가 하나도 없으면 에러 반환: `"이번 주 작성된 데일리 노트가 없습니다"`
- Gemini 호출 → 결과를 마크다운 파일로 저장
- 응답: `{ path, content }` (저장된 파일 경로 + 생성된 리뷰 내용)

## 주차 계산

- ISO 8601 기준 주차 사용 (월요일 시작)
- 파일명 예시: `2026-W14.md` (2026년 14주차)
- 해당 주의 날짜 범위: 월요일~일요일

## 프롬프트 설계

```
[시스템 지시]
당신은 개인 생산성 코치입니다. 아래 데일리 노트에서 인사이트를 추출하세요.

규칙:
- 불릿 포인트만 출력 (서론, 결론, 인삿말 없이)
- 반복되는 주제, 감정 변화, 숨은 패턴, 주목할 만한 점 중심
- 이전 주 리뷰가 있다면 참고하여 연속성 있는 관찰을 포함
- 한국어로 작성

[이전 주 리뷰] (있으면)
(이전 주 리뷰 내용)

[이번 주 데일리 노트]
--- 월요일 (YYYY-MM-DD) ---
(노트 내용)

--- 화요일 (YYYY-MM-DD) ---
(노트 내용)

...
```

## 저장 파일 형식

```markdown
# 2026년 14주차 (3/30 ~ 4/5)

- 인사이트 1
- 인사이트 2
- 인사이트 3
...
```

- 제목에 주차 번호와 날짜 범위를 표시
- 본문은 Gemini 출력(불릿 포인트)을 그대로 저장

## 자동 실행

- 매주 일요일 밤 cron 스케줄로 API 호출
- UI에 수동 실행 버튼 없음

## 이전 주 리뷰 참조

- `data/notes/weekly-review/` 폴더에서 파일명 기준으로 직전 주차 파일을 찾음
- 파일이 없으면 (첫 실행 등) 이전 주 리뷰 없이 진행
- 이를 통해 LLM이 연속성 있는 인사이트를 뽑을 수 있음
  - 예: "지난주에도 언급됐던 X가 이번 주에도 반복되고 있다"

## 기존 시스템과의 통합

- **범용 노트 UI**: `weekly-review` 폴더가 파일 브라우저에 자동으로 표시됨 (추가 UI 작업 불필요)
- **AI 브리핑**: 브리핑과 독립적으로 동작, 겹치지 않음
- **저장소**: 기존 `file-store.ts`의 파일 관리 체계를 그대로 활용

## 에러 처리

- 데일리 노트가 하나도 없으면 리뷰를 생성하지 않음
- Gemini 호출 실패 시 재시도 없이 에러 로그만 남김
- 이미 해당 주차 리뷰가 존재하면 덮어쓰기

## 범위 밖 (YAGNI)

- 수동 실행 UI
- 할 일/D-day 데이터 참조
- 주간 리뷰끼리 장기 트렌드 분석
- 리뷰 수정/재생성 UI (파일 직접 편집으로 충분)
- 리뷰 삭제 보호 (범용 노트 UI의 삭제 기능 그대로 사용)
